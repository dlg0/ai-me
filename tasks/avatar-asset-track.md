# Avatar Asset Track

This workstream runs in parallel with the controller. Follow `docs/ASSET_PIPELINE.md` and `docs/RIGGING_SPEC.md`.

## A-001 — Select source avatar and record rights/provenance

- [ ] source file selected;
- [ ] highest-resolution/layered version obtained;
- [ ] adaptation and video/workplace usage rights confirmed;
- [ ] desired segmented/stylised direction selected;
- [ ] source manifest created outside or inside approved private storage.

## A-002 — Layer decomposition/reconstruction

- [ ] facial/body layers separated;
- [ ] hidden geometry repainted;
- [ ] masks/draw order documented;
- [ ] neutral layered source reviewed;
- [ ] no private/restricted asset accidentally committed.

## A-003 — Minimum Live2D rig

- [ ] stable head X/Y/Z;
- [ ] blink/slow blink;
- [ ] brow and gaze sufficient for uncertainty;
- [ ] neutral/mild agreement/thinking/uncertain/boundary expressions;
- [ ] speech-ready/qualified-answer expression;
- [ ] nod and head-tilt actions;
- [ ] reset-neutral stable and repeatable.

## A-004 — VTube Studio configuration and rig profile

- [ ] VTube inputs mapped to Live2D parameters;
- [ ] required hotkeys named uniquely;
- [ ] hotkeys/input parameters visible through API preflight;
- [ ] `rig-profile.v1` created and validates;
- [ ] installation notes and model version recorded.

## A-005 — Visual acceptance

- [ ] expression contact sheet;
- [ ] no holes/tearing at expected ranges;
- [ ] key states legible at Teams-tile size;
- [ ] physics restrained;
- [ ] profile and model reset together;
- [ ] ready for M1-009 recording.
